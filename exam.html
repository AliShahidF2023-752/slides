Week 1: Web Fundamentals & Architecture

The Client-Server Model

The web works on a request-response cycle.

  • Client: The browser (Chrome, Safari). Initiates requests.
  • Server: The computer storing the website. Processes requests and sends responses.
  • Protocol (HTTP): The language they speak (HyperText Transfer Protocol).

Core Technologies

Technology Role Analogy
HTML Structure & Content The Skeleton
CSS Presentation & Style The Skin/Clothes
JavaScript Behavior & Interactivity The Muscles/Brain

Week 2: HTML5 Structure & Semantics

Semantic vs. Non-Semantic

Semantic HTML means using tags that describe their meaning to both the browser and the developer.

  • Non-Semantic: <div>, <span> (Tells you nothing about content)
  • Semantic: <header>, <nav>, <article>, <footer> (CLEAR meaning)

Why use Semantic? SEO (Search engines understand better), Accessibility (Screen readers), and Maintainability.

Block vs. Inline Elements

  • Block: Starts on a new line, takes full width. Examples: <div>, <p>, <h1>, <section>.
  • Inline: Takes only necessary width, no new line. Examples: <span>, <a>, <img>, <strong>.

Week 3: CSS Styling & Layouts

The Box Model

Every element in web design is a rectangular box.

+---------------------------+
|        Margin             |
|  +---------------------+  |
|  |     Border          |  |
|  |  +---------------+  |  |
|  |  |   Padding     |  |  |
|  |  |  +---------+  |  |  |
|  |  |  | Content |  |  |  |
|  |  |  +---------+  |  |  |
|  |  +---------------+  |  |
|  +---------------------+  |
+---------------------------+

Calculated Width: Content + Padding + Border (if box-sizing: content-box).
Best Practice: Use * { box-sizing: border-box; } so padding doesn't increase width.

Centering in CSS

The age-old problem. Solutions:

  • Text: text-align: center;
  • Block Element: margin: 0 auto; (requires width)
  • Modern Way (Flexbox): 
    .parent {
    display: flex;
    justify-content: center; /* Horizontal */
    align-items: center;     /* Vertical */
}

Week 4: Mobile-First & Flexbox

Mobile-First Design Strategy

Designing for the smallest screen first, then enhancing for larger screens.

  • Why? Performance (load less code for mobile), User Base (more mobile users), Focus (prioritize content).
  • Technique: Use min-width media queries.
/* Base styles = Mobile */
.container { width: 100%; }

/* Tablet */
@media (min-width: 768px) {
    .container { width: 80%; }
}

Flexbox Cheat Sheet

  • display: flex; - Activates flexbox.
  • flex-direction: row | column; - Main axis direction.
  • justify-content: flex-start | center | space-between; - Main axis alignment.
  • align-items: stretch | center | flex-start; - Cross axis alignment.
  • flex-wrap: wrap; - Allow items to wrap to next line.
  • gap: 10px; - Space between items.

Week 5: JavaScript & Node.js Fundamentals

Variables & Data Types

  • const: Cannot be reassigned. Use by default.
  • let: Can be reassigned. Use for counters, state.
  • var: Old function-scoped. Avoid.

Array Methods (Higher Order)

Modern JS relies heavily on these methods:

  • .map(): Transform each element. Returns new array. (React lists)
  • .filter(): Remove elements based on condition. Returns new array.
  • .forEach(): Loop through. Returns nothing.
  • .find(): Return first matching item.

Node.js Concept

Node.js is a runtime that lets you run JavaScript on the server (outside the browser). It uses the V8 engine.

Week 6: Backend APIs (Express.js)

What is an Endpoint?

Endpoint = URL + Method. It's a "door" to your server.

  • GET: Retrieve data.
  • POST: Create data.
  • PUT/PATCH: Update data.
  • DELETE: Remove data.

HTTP Status Codes

Range Meaning Examples
2xx Success 200 OK, 201 Created
4xx Client Error 400 Bad Request, 404 Not Found
5xx Server Error 500 Internal Server Error

Express Code Example

const express = require('express');
const app = express();

app.get('/api/users', (req, res) => {
    res.json([{ name: 'Ali' }, { name: 'Hamad' }]);
});

app.listen(3000);

Week 7: React Basics & JS Equality

Strict Equality (===) vs Loose (==)

Always use === (Strict) to avoid type coercion bugs.

  • 5 == "5" is TRUE (Bad)
  • 5 === "5" is FALSE (Good - Check Type)

React Concepts

  • JSX: HTML-like syntax inside JS.
  • Components: Reusable UI blocks (Functions returning JSX).
  • Props: Data passed down to components (Read-only).
  • State: Data managed inside a component (Changeable).

Week 9: Digital Marketing (SEO & UX)

SEO (Search Engine Optimization)

The practice of optimizing web pages to rank higher in search results.

  • On-Page: Content, Tags, Speed.
  • Off-Page: Backlinks, Social signals.
  • Technical: Sitemap, Robots.txt, HTTPS.

UX (User Experience)

Core Principle: Don't make the user think.

Usability Heuristics: Visibility of system status, Match between system and real world, User control and freedom.

Week 12: Case Study - CareDesk

Project Overview

A Hospital Management System built with Next.js, Supabase, and AI Assistance (Cursor/Claude).

Key Learnings

  • AI Coding: Using prompts to generate boilerplate and complex logic.
  • Database: Using Supabase (PostgreSQL) for relational data.
  • RLS (Row Level Security): Ensuring users only see their own data.

System Design & SDLC

SDLC Phases (Waterfall/Agile)

  1. Planning: Feasibility, timeline, cost.
  2. Analysis (Requirements): Gathering what the system must do.
  3. Design: Architecture, UI/UX, database schema.
  4. Implementation (Coding): Writing the code.
  5. Testing: Verifying it works (Unit, Integration, UAT).
  6. Deployment: Releasing to production.
  7. Maintenance: Updates and fixes.

SRS & Requirements (IEEE 830)

Functional vs. Non-Functional

  • Functional (FR): What the system does. (e.g., "User can login", "System sends email").
  • Non-Functional (NFR): How the system performs (Constraints). (e.g., Speed, Security, Reliability).

Characteristics of Good Requirements

Use "Shall" for mandatory reqs.

  • Correct
  • Unambiguous (Only one interpretation)
  • Complete
  • Consistent
  • Verifiable (Testable) - "Load in 2 seconds" vs "Load fast"

Final Exam Specific Topics

Responsive Design

Viewport Meta Tag: Critical for mobile. width=device-width, initial-scale=1.0

Media Queries: @media (max-width: 768px) { ... }

Analytics (Microsoft Clarity)

  • Heatmaps: Visualize where users click/scroll.
  • Dead Clicks: Clicking on non-interactive elements (Bad UX).
  • Rage Clicks: Rapidly clicking frustration.

Short Answer Questions

1. Explain the Mobile-First approach.

Answer: Designing for the smallest screen (mobile) first and then using min-width media queries to add styles for larger screens (tablets, desktops). This ensures better performance on mobile devices.

2. Difference between margin and padding?

Answer: Margin is space outside the border (pushes other elements away). Padding is space inside the border (between border and content).

3. Why use 'key' in React lists?

Answer: Keys help React identify which items have changed, added, or removed. It optimizes rendering performance by updating only changed items.

4. Explain the 'useEffect' dependency array.

Answer: The array passed as the second argument controls when the effect runs. [] = runs once (mount). [prop] = runs when prop changes. No array = runs every render.

5. What is Semantic HTML?

Answer: Using HTML tags that convey meaning about the enclosed content (e.g., <article> vs <div>). Aids accessibility and SEO.
